## 🫧 1. Bubble Sort

> 💡 *แนวคิด: เปรียบเทียบทีละคู่ ดันค่ามากสุดไปท้ายเหมือนฟองลอย*

### 🔹 คอนเซปต์หลัก
วิธีนี้อาศัยการเปรียบเทียบข้อมูลที่อยู่ติดกันทีละคู่ โดยเริ่มจากซ้ายสุด ถ้าระบุให้เรียงจากน้อยไปมาก หากตัวซ้ายมีค่ามากกว่าตัวขวา จะทำการสับเปลี่ยน (Swap) ตำแหน่งกัน จากนั้นขยับไปเปรียบเทียบคู่ถัดไปทางขวาเรื่อยๆ จนจบแถว เมื่อจบรอบ ค่าที่มากที่สุดจะถูกดันไปอยู่ขวาสุดเหมือนฟองอากาศที่ลอยขึ้นผิวน้ำ จากนั้นก็ทำซ้ำในรอบถัดไปโดยไม่ต้องเช็คตำแหน่งสุดท้ายที่เรียงเสร็จแล้ว  

### 🔹 จุดเด่นและข้อสังเกต
- ⚪ เข้าใจง่ายที่สุด แต่ทำงานได้ช้า  
- ⚪ ไม่เหมาะกับข้อมูลขนาดใหญ่ เหมาะกับข้อมูลจำนวนน้อยๆ  

<details>
<summary>💻 ดูโค้ด Python</summary>

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
    return arr
```
</details>

## 🎯 2. Selection Sort

> 💡 *แนวคิด: หา “ค่าน้อยสุด” แล้วเอามาวางทีละตำแหน่ง*

### 🔹 คอนเซปต์หลัก
แบ่งข้อมูลออกเป็น 2 ส่วนในใจ คือส่วนที่ "เรียงแล้ว" กับ "ยังไม่เรียง" ในแต่ละรอบ อัลกอริทึมจะวิ่งหาค่าที่ "น้อยที่สุด" จากในกลุ่มที่ยังไม่ได้เรียง เมื่อเจอแล้ว จะนำค่านั้นมาสับเปลี่ยน (Swap) กับข้อมูลตัวแรกสุดของกลุ่มที่ยังไม่ได้เรียง ทำแบบนี้ซ้ำไปเรื่อยๆ จนกว่าข้อมูลจะถูกย้ายไปอยู่ฝั่งที่เรียงแล้วทั้งหมด  

### 🔹 จุดเด่นและข้อสังเกต
- ⚪ ช่วยลดจำนวนครั้งในการสับเปลี่ยน (Swap) ข้อมูลได้ดี แต่จำนวนการเปรียบเทียบยังคงสูงอยู่  
- ⚪ ในกรณีทั่วไป (Average case) ทำงานได้ดีกว่า Bubble sort แต่ในกรณีดีที่สุด (Best case) จะทำงานแย่กว่า  

<details>
<summary>💻 ดูโค้ด Python</summary>

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i+1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```
</details>

## 🧩 3. Insertion Sort

> 💡 *แนวคิด: แทรกค่าทีละตัวในตำแหน่งที่ถูกต้อง*

### 🔹 คอนเซปต์หลัก
แบ่งข้อมูลเป็น 2 ส่วน (เรียงแล้ว/ยังไม่เรียง) เช่นกัน แต่ใช้วิธีหยิบข้อมูลตัวแรกสุดจากกลุ่มที่ "ยังไม่เรียง" มาหาตำแหน่งที่เหมาะสมในกลุ่มที่ "เรียงแล้ว" เมื่อเจอตำแหน่งที่ถูกต้อง จะทำการคัดลอกและขยับ (Copy/Shift) ข้อมูลตัวอื่นๆ ไปทางขวาเพื่อเว้นที่ว่าง แล้วแทรกข้อมูลนั้นลงไป (จะไม่มีการ Swap แบบสองวิธีแรก)  

### 🔹 จุดเด่นและข้อสังเกต
- ⚪ ไม่มีการ Swap ข้อมูล ใช้แค่การ Copy เท่านั้น  
- ⚪ ทำงานได้เร็วกว่า Bubble Sort ถึง 2 เท่า และเร็วกว่า Selection Sort ด้วย  
- ⚪ เป็นวิธีที่ใช้งานได้หลากหลายและดีที่สุดใน 3 ตัวนี้ โดยเฉพาะถ้าข้อมูลมีขนาดเล็กหรือค่อนข้างเรียงลำดับมาอยู่แล้วบ้าง  

<details>
<summary>💻 ดูโค้ด Python</summary>

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and key < arr[j]:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```
</details>

## 📈 4. Complexity Analysis

### ⏱️ Time Complexity

| Algorithm        | 🟢 Best Case | 🟡 Average Case | 🔴 Worst Case |
|------------------|-------------|----------------|---------------|
| Bubble Sort      | $\Omega(n)$ | $\theta(n^2)$  | $O(n^2)$      |
| Selection Sort   | $\Omega(n^2)$ | $\theta(n^2)$ | $O(n^2)$      |
| Insertion Sort   | $\Omega(n)$ | $\theta(n^2)$  | $O(n^2)$      |

### 💾 Space Complexity
ทั้ง 3 อัลกอริทึมใช้พื้นที่หน่วยความจำเพิ่มเติมเพียง $O(1)$ ทำให้เมื่อรวมกับตัว Array ข้อมูลเริ่มต้นแล้ว จะมี Space Complexity รวมอยู่ที่ $O(n)$
