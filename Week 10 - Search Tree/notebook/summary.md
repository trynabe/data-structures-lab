# 🧠 1. Introduction to Trees

## 🔹 แนวคิดหลัก

Tree เป็นโครงสร้างข้อมูลแบบไม่เป็นเชิงเส้น (Nonlinear) ที่ใช้จัดระเบียบข้อมูลตามลำดับชั้น (Hierarchical) ซึ่งแตกต่างจาก Array หรือ Linked List

## 🔹 จุดเด่น

มีอัลกอริทึมในการค้นหาและดึงข้อมูลที่มีประสิทธิภาพสูงกว่าโครงสร้างข้อมูลแบบเชิงเส้น

## 🔹 การใช้งานจริง

* โครงสร้างระบบไฟล์ (File systems)
* เมนูในหน้าจอผู้ใช้ (GUI elements)
* โครงสร้างองค์กร

โครงสร้างของ Tree มักใช้แสดงความสัมพันธ์แบบเครือญาติ (Family Tree) หรือโครงสร้างการสืบทอดคลาสใน Python (Class Hierarchy)

---

# 📚 2. Terminology in Trees

## 🔹 Node & Edge

ข้อมูลจะถูกเก็บไว้ใน Node และเส้นที่เชื่อมความสัมพันธ์ระหว่าง Parent (แม่) และ Child (ลูก) เรียกว่า Edge

## 🔹 คำศัพท์สำคัญ

* **Root (ราก)** → Node บนสุดของต้นไม้ ซึ่งจะไม่มี Parent
* **Leaf (ใบ)** → Node ที่อยู่ปลายสุดและไม่มี Child
* **Internal Node** → Node ที่อยู่ตรงกลางและมี Child อย่างน้อย 1 ตัว
* **Siblings** → Node ที่มี Parent ตัวเดียวกัน
* **Path** → ลำดับของ Node ที่มี Edge เชื่อมต่อกัน

---

# 📏 3. Depth & Height

## 🔹 Depth (ความลึกของ Node)

นับจำนวนบรรพบุรุษ (Ancestors) ทั้งหมดของ Node นั้นๆ (ไม่รวมตัวมันเอง) โดยคำนวณแบบ Recursion คือ

* ถ้าเป็น Root จะมี Depth เป็น 0
* ถ้าไม่ใช่ จะเท่ากับ 1 + Depth ของ Parent

## 🔹 Height (ความสูงของ Node/ต้นไม้)

นับจาก Depth ที่ลึกที่สุดของ Leaf ที่อยู่ใต้ตัวมัน

การคำนวณความสูงที่มีประสิทธิภาพควรใช้ Recursion ทางตรงซึ่งจะใช้เวลาในการประมวลผล (Complexity) แบบ $O(n)$ เพื่อหลีกเลี่ยงวิธีคำนวณซ้ำซ้อนที่กินเวลา $O(n^{2})$

---

# 🧩 4. Tree Abstract Data Type (ADT)

## 🔹 Tree ADT

เป็นการกำหนดกรอบและระเบียบวิธีที่ใช้จัดการโครงสร้างต้นไม้ โดยนิยามตัวแปร Position ขึ้นมาเพื่อเป็นตัวแทนของ Node

## 🔹 เมธอดพื้นฐาน

* `T.root()` → หา Root
* `T.parent(p)` → หา Parent
* `T.num_children(p)` → หาจำนวนลูก
* `T.is_leaf(p)` → เช็กว่าเป็น Leaf ไหม
* `T.is_empty()` → เช็กว่าต้นไม้ว่างเปล่าไหม

## 🔹 การเขียนใน Python

สร้างเป็น Abstract Base Class โดยใช้หลักการ Duck Typing ซึ่งบางเมธอดจะถูกบังคับให้เขียนในคลาสลูก (Subclass) ด้วยการโยน Error ชนิด NotImplementedError

---

# 🔗 5. Linked Structure (Realization)

## 🔹 การจัดการหน่วยความจำ

นิยมใช้การเชื่อมโยง (Linked Structure) แทนเพราะให้ความยืดหยุ่นในการจัดสรรหน่วยความจำและอัปเดตข้อมูลได้ง่าย

## 🔹 โครงสร้างของแต่ละ Node

จะจัดเก็บ 3 อย่าง คือ

* ข้อมูล (Element)
* ตัวอ้างอิงไปยัง Parent
* List สำหรับเก็บตัวอ้างอิงของ Child

## 🔹 Position Wrapper

ในการเข้าถึง Tree อย่างปลอดภัย จะมีการใช้ Position เพื่อห่อหุ้ม Node เอาไว้ ช่วยป้องการเข้าถึงโครงสร้างข้อมูลโดยตรง (Encapsulation)

---

# ⚙️ 6. Tree Operations

## 🔹 การเพิ่มข้อมูล (add_child)

เป็นการสร้าง Node ใหม่พร้อมกับระบุให้ไปเชื่อมกับ Parent ที่ต้องการ

## 🔹 การลบข้อมูล (remove)

เมื่อลบ Node หนึ่งๆ อัลกอริทึมแบบวนซ้ำ (Recursion) จะทำงานเพื่อลบ Node ลูกหลาน (Descendants) ที่อยู่ภายใต้ตัวมันออกทั้งหมด

## 🔹 การค้นหา (find_element)

ใช้กระบวนการค้นหาแบบ Depth-First Search (DFS) โดยถ้าเจอข้อมูลที่ต้องการจะคืนค่าตัวแทน (Position) กลับมา แต่ถ้าหาไม่เจอในต้นไม้จะคืนค่าเป็น None

# 🧪 7. Testing

## 🔹 ความสำคัญ

การทดสอบโค้ดจำเป็นอย่างมาก เพื่อให้มั่นใจว่าการเพิ่ม, การลบ, การค้นหา และการรักษาโครงสร้างของต้นไม้ทำงานได้ถูกต้องและไม่มีบั๊กซ่อนอยู่

## 🔹 กระบวนการทดสอบ

มีการทดสอบ:

* ความถูกต้องของโครงสร้าง (Structure)
* ความสัมพันธ์ของแม่-ลูก (Parent references)
* การเพิ่ม Node ย่อย (Add child)
* การลบ (Remove)
* การดึงข้อมูล (Find element)
