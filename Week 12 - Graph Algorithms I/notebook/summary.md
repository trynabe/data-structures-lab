# 🌐 1. ความรู้เบื้องต้นเกี่ยวกับกราฟ (Graphs)

> 💡 **ทริคช่วยจำ:** ให้นึกภาพกราฟเหมือน **"แผนที่สถานีรถไฟฟ้า"** ครับ

- 🎯 **V (Vertices/Nodes):** คือ "ตัวสถานีรถไฟฟ้า" แต่ละสถานี  
  เช่น สถานีสยาม, สถานีอโศก
- 🚆 **E (Edges/Arcs):** คือ "รางรถไฟ" ที่เชื่อมระหว่างสถานี

> 📝 **สรุป:** กราฟไม่ใช่แผนภูมิ (Chart) เช่น กราฟแท่งหรือแผนภูมิวงกลม แต่กราฟในทางคอมพิวเตอร์คือโครงสร้างที่ใช้เอาข้อมูลมาเขียนโยงกัน เพื่อดูว่า "ความสัมพันธ์" หรืออะไรเชื่อมกับอะไรบ้างนั่นเองครับ

## 🔀 2. ประเภทของกราฟและเส้นเชื่อม (Types of Graphs & Edges)

เปรียบเทียบความสัมพันธ์ของเส้นเชื่อมและกราฟประเภทต่างๆ ดังนี้ครับ:

- 📱 **Directed Graph (มีทิศทาง):** เหมือนระบบ *"Follower ใน X (Twitter)"*  
  การที่ A ติดตาม B ไม่ได้แปลว่า B ต้องติดตาม A กลับ (เป็นเส้นทางเดินรถทางเดียว)

- 👥 **Undirected Graph (ไร้ทิศทาง):** เหมือนระบบ *"เพื่อนใน Facebook"*  
  ถ้า A เป็นเพื่อนกับ B แสดงว่า B ก็เป็นเพื่อนกับ A ด้วย (เดินสวนเลนกันได้)

- 🔀 **Mixed Graph (กราฟแบบผสม):** มีเส้นเชื่อมทั้งสองแบบผสมกัน

- 📊 **Weighted Graphs (ถ่วงน้ำหนัก):** เหมือนแผนที่ *"Google Maps"*  
  เส้นทางแต่ละสายมี "ระยะทาง", "ค่าใช้จ่าย" หรือ "เวลา" (น้ำหนัก) กำกับไว้บนเส้นนั้น

## 📚 3. คำศัพท์ที่ควรจำสำหรับสอบ (Graph Terminology)

จัดกลุ่มคำศัพท์ให้จำง่ายๆ สำหรับนำไปสอบครับ:

### 🔢 กลุ่มบอกจำนวน (Degree)

| คำศัพท์ | ความหมาย | ตัวอย่าง |
| :---: | :--- | :--- |
| **Degree** | จำนวนเส้นเชื่อมที่เชื่อมต่อโหนดนั้น | มีถนนเชื่อมกี่สาย |
| **Indegree** | เส้นเชื่อมที่พุ่งเข้า | มีคนพุ่งเข้าหาเรากี่คน |
| **Outdegree** | เส้นเชื่อมที่พุ่งออก | เราพุ่งออกไปหาคนอื่นกี่คน |

### 📍 กลุ่มโหนดพิเศษ

| 🏷️ ชื่อ | ความหมาย | Degree |
| :---: | :--- | :---: |
| **Leaf / Pendant Vertex** | ทางตัน - มีเส้นเชื่อมแค่เดียว | 1 |
| **Isolated Vertex** | เกาะร้าง - ไม่มีเส้นเชื่อม | 0 |
| **Source Vertex** | จุดปล่อยน้ำ - มีแต่ลูกศรพุ่งออก | Indegree = 0 |
| **Sink Vertex** | หลุมดำ - มีแต่ลูกศรพุ่งเข้า | Outdegree = 0 |

### 🛣️ กลุ่มลักษณะเส้นเชื่อมและเส้นทาง

- **Adjacency (การประชิด):** โหนด 2 โหนดประชิดกันเมื่อมีเส้นเชื่อมต่อถึงกันโดยตรง
- **Parallel Edges (พหุเอดจ์):** เส้นเชื่อมตั้งแต่ 2 เส้นขึ้นไปเชื่อมระหว่างจุดเดียวกัน
- **Self Loop (ลูปอัตตา):** เส้นเชื่อมที่วนกลับมาที่โหนดเดิม
- **Simple Graph (กราฟเชิงเดี่ยว):** ไม่มี Parallel edges และไม่มี Self loops
- **Path (วิถี):** ลำดับการเดินจากจุด A ไปจุด B ตามเส้นที่เชื่อมต่อ
  - *Simple Path:* วิถีที่ไม่เดินซ้ำโหนด
- **Cycle (วัฏจักร/วง):** วิถีที่จุดเริ่มต้นและสิ้นสุดเป็นโหนดเดียวกัน

### 🌳 กลุ่มต้นไม้ (Tree)

- **Tree:** กราฟที่เชื่อมกันหมด แต่ **ห้ามมีวง (Cycle)** เด็ดขาด
- **Forest:** กราฟที่ไม่มีวง = Tree หลายๆ ต้นรวมกัน
- **Spanning Tree:** ต้นไม้ที่เป็นกราฟย่อยมีโหนดครบทุกตัวในกราฟตั้งต้น

## ⚡ 4. ทฤษฎีบทและสูตรที่สำคัญ (Propositions & Formulas)

หมวดนี้มักออกสอบคำนวณหรือทฤษฎีครับ:

📌 **ทฤษฎีบทการจับมือ** (ผลรวมองศา = 2m)
- ในกราฟไร้ทิศทาง: เส้นเชื่อม 1 เส้นแตะโหนด 2 โหนด
- ∑ Degree = 2 × จำนวนเส้นเชื่อม

📌 **ผลรวมของ Indegree และ Outdegree**
- ในกราฟมีทิศทาง: ∑ Indegree = ∑ Outdegree = m (จำนวนเส้นเชื่อม)

📌 **เส้นเชื่อมสูงสุด (Maximum Edges):** $O(n^2)$
- แต่ละโหนด n สามารถลากเส้นไปหาเพื่อนได้ n-1 โหนด
  - **Undirected:** $\frac{n(n-1)}{2}$ (หารด้วย 2 เพื่อไม่ซ้ำ)
  - **Directed:** $n(n-1)$

📌 **สูตรเด็ดของ Tree vs Connected Graph**

| ประเภท | เงื่อนไข |
| :---: | :--- |
| **Connected Graph** | $m \geq n - 1$ (ต่ออย่างน้อย) |
| **Tree** ⭐ | $m = n - 1$ (เด็ดขาด!) |
| **Forest** | $m \leq n - 1$ |

> ⚠️ **กฎเหล็ก Tree:** ถ้า m > n-1 → มี Cycle  
> ถ้า m < n-1 → ขาดตอน (Disconnected)

## 💾 5. โครงสร้างข้อมูลที่ใช้สร้างกราฟ (Graph Data Structures)

ในทางโปรแกรมมิ่ง เราต้องเก็บกราฟลงคอมพิวเตอร์ ทำได้ 2 แบบหลักๆ:

| 🏗️ คุณสมบัติ | 📋 Adjacency List (รายการประชิด) | 🔢 Adjacency Matrix (เมทริกซ์ประชิด) |
| :---: | :--- | :--- |
| **ลักษณะ** | 📖 สมุดหน้าเหลือง จดเฉพาะเส้นที่มี | 🎲 ตารางหมากรุก (2D) 1 = เชื่อม, 0 = ไม่เชื่อม |
| **ตัวอย่าง** | `{A: [B,C], B: [A]}` | ตาราง n×n |
| **ความเร็ว** | ✅ ดีเมื่อหาเพื่อน | ✅ ดีเมื่อตรวจสอบเชื่อม |
| **พื้นที่ (Memory)** | ⭐ ประหยัด = $O(n+m)$ | ⭐⭐ กินพื้น = $O(n^2)$ |
| **เหมาะสำหรับ** | Sparse Graph (เส้นน้อย) | Dense Graph (เส้นเยอะ) |

## 🔍 6. การแวะผ่านกราฟ (Graph Traversals) 🎯 **จุดเน้นสำคัญมากๆ!**

การแวะผ่านกราฟคือการเดินทางไปตรวจสอบทุกๆ โหนด เปรียบเทียบ 2 วิธีหลัก:

| 🚀 วิธี | 📸 การทำงานเปรียบเทียบ | 🛠️ ข้อมูล | ✨ ประโยชน์หลัก |
| :---: | :--- | :--- | :--- |
| **BFS (แนวกว้าง)**<br>🌊 *Breadth-First* | 💧 **"โยนหินลงน้ำ"** - คลื่นแผ่ออกเป็นวงกว้าง สำรวจพี่น้อง (Level เดียว) ให้ครบก่อน ค่อยลงลึก | **Queue**<br>(FIFO) | 🎯 หา **Shortest Path** ในกราฟไม่ถ่วงน้ำหนัก |
| **DFS (แนวลึก)**<br>🕳️ *Depth-First* | ⛰️ **"เดินเขาวงกต"** - เดินลงลึกสุดก่อน ถ้าตันก็ถอยหลัง (Backtrack) หาเส้นอื่น | **Stack / Recursion**<br>(LIFO) | 🌳 Topological sort, Cycle detect, DFS-tree |

### 🔬 ความซับซ้อน (Time & Space Complexity)

```
ทั้ง BFS และ DFS:
⏱️  เวลา:  O(V + E)  → V = จำนวนโหนด, E = จำนวนเส้นเชื่อม
💾 พื้นที่: O(V)      → เก็บ visited + queue/stack
```

### 📊 เปรียบเทียบรายละเอียด

| 🔎 ลักษณะ | BFS 🌊 | DFS ⛰️ |
| :---: | :---: | :---: |
| **ลำดับการเยี่ยม** | ระดับ 0 → ระดับ 1 → ระดับ 2 | ไปลึกสุดก่อน ถอยกลับ |
| **โครงสร้าง Output** | Shortest path tree | DFS tree / Forest |
| **ใช้สำหรับ** | Navigation, Social network | Backtracking, Puzzles |
| **ตัวอย่างจริง** | Google Maps (ระยะทาง) | Maze solving, Sudoku |

## 🎯 7. การประยุกต์ใช้และอัลกอริทึมคลาสสิก (Real-World Applications & Algorithms)

กราฟมีประโยชน์มากในการจำลองปัญหาโลกจริง เช่น:
- 🌐 เครือข่ายคอมพิวเตอร์
- 📍 ระบบนำทาง GPS
- 👥 เครือข่ายสังคมออนไลน์
- 🚗 Logistics & Delivery
- ⚡ Power Grid Distribution

### 🏆 ปัญหาคลาสสิก 5 ข้อที่ต้องรู้:

#### 1️⃣ **Shortest Path Problem** 🛤️
- 🎯 **หาเส้นทางสั้นสุด**
- 💼 ตัวอย่าง: Grab/Uber หา route เร็วสุด
- 🔧 อัลกอริทึม: **Dijkstra**, Bellman-Ford
- 📊 ใช้เมื่อ: กราฟถ่วงน้ำหนักบวก

#### 2️⃣ **Minimum Spanning Tree (MST)** 🌳
- 🎯 **หาต้นไม้ทอดข้ามที่น้ำหนักรวมน้อยสุด**
- 💼 ตัวอย่าง: การไฟฟ้า ลากสายให้ครอบคลุม ใช้สายอย่างน้อย
- 🔧 อัลกอริทึม: **Kruskal**, **Prim**
- 📊 สูตร: MST มี (n-1) เส้นเชื่อม

#### 3️⃣ **Traveling Salesman Problem (TSP)** 🧳
- 🎯 **หาเส้นทางแวะทุกโหนดแล้วกลับจุดเริ่มต้น ให้สั้นสุด**
- 💼 ตัวอย่าง: Courier ต้องส่งของ 10 ที่แล้วกลับบริษัท วิ่งระยะรวมน้อยสุด
- ⚠️ **โปรแกรมกึ่ง (NP-Hard)** - ยากมาก!
- 🔧 วิธีแก้: Brute force (เล็ก), Heuristics (ใหญ่)

#### 4️⃣ **Maximum Network Flow** 💧
- 🎯 **หาปริมาณการไหลในเครือข่ายให้สูงสุด**
- 💼 ตัวอย่าง: ระบบท่อน้ำเมือง ส่งน้ำปริมาณสูงสุดโดยท่อไม่แตก
- 🔧 อัลกอริทึม: **Ford-Fulkerson**

#### 5️⃣ **Topological Sort** 📋
- 🎯 **เรียงลำดับโหนด ตามลำดับการทำงาน**
- 💼 ตัวอย่าง: Task scheduling, Build dependencies
- 🔧 ใช้: **DFS**
- 📊 เฉพาะ **DAG** (Directed Acyclic Graph) เท่านั้น!

## 📌 **สรุป Quick Reference Guide**

| 🎯 ปัญหา | 🔧 อัลกอริทึม | ⏱️ Complexity | 📌 เงื่อนไข |
| :---: | :---: | :---: | :--- |
| Shortest Path | Dijkstra | $O((V+E)\log V)$ | Positive weights |
| MST | Kruskal/Prim | $O(E \log E)$ | Connected |
| Topological Sort | DFS | $O(V+E)$ | DAG only |
| Max Flow | Ford-Fulkerson | $O(VE^2)$ | Flow network |
| Cycle Detection | DFS/BFS | $O(V+E)$ | All graphs |
